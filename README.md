# UV Index App

Live UV index for your location. Flutter, Android-first.

## Features

- Color-coded UV dial (WHO bands: Low/Moderate/High/Very High/Extreme) with
  safety advice and an estimated time-to-burn, scaled by your Fitzpatrick
  skin type.
- "Safe to go outside" / "protection needed again after …" windows, based on
  a configurable safe threshold.
- Scrollable 24-hour forecast histogram.
- Home-screen widget (1x1) showing the current index, refreshed in the
  background on a configurable interval.
- Local notifications when UV crosses into the "High" band, and again when
  it drops back to a safe level — each independently toggleable.
- Manual location override via city/address search, for when you don't want
  to use the device's GPS.
- Works offline: falls back to the last successfully fetched reading and
  marks it "Offline" if a refresh fails.
- Localized in English and French, following the device's system locale
  automatically (no in-app language switcher).

## First-time setup

You need Flutter installed (https://docs.flutter.dev/get-started/install)
and Android Studio with an emulator or a physical device.

1. Install dependencies (this also generates the localization classes under
   `lib/l10n/`, since `generate: true` is set in `pubspec.yaml`):
   ```
   flutter pub get
   ```

2. Run it:
   ```
   flutter run
   ```

## How it works

- On launch it uses the manual location override if one is set, otherwise
  requests coarse GPS location, then calls `currentuvindex.com/api/v1/uvi`
  (no API key needed).
- The first load shows a full-screen spinner; subsequent refreshes (pull-to-
  refresh, or returning from Settings) keep the last data on screen and show
  a thin progress bar instead, so the screen never goes blank.
- If the network or location fails, it falls back to the last cached
  reading and marks it "Offline".
- A background task (`workmanager`) periodically re-fetches the UV index
  and updates the home-screen widget and notifications even when the app
  isn't open.

## Localization

UI strings live in `lib/l10n/app_en.arb` (template) and `lib/l10n/app_fr.arb`.
To add a string: add the key to `app_en.arb` first, mirror it in every other
`app_*.arb` file, then run `flutter gen-l10n` (or `flutter pub get`) to
regenerate `AppLocalizations`. To add a new language, copy `app_en.arb` to
`app_<code>.arb`, translate the values, and add an `android/app/src/main/res/values-<code>/strings.xml`
for the widget/manifest strings.

`notification_service.dart` is the one exception: it runs in a context-free
background isolate (no `BuildContext`), so it can't use the generated
`AppLocalizations` and instead checks `ui.PlatformDispatcher.instance.locale`
directly with inline EN/FR strings.

## Notes & next steps

- **Time-to-burn** is a rough display heuristic, not medical advice.
- **API**: free and keyless, rate-limited by IP. Fine for development. Before
  a heavy Play Store launch, consider OpenWeatherMap/OpenUV with a key for
  stable commercial terms.
